# P3 功能路线图 — v2.0 规划

> 以下功能需要云端后端或较大基础设施投入，规划在 v2.0 版本实现。
> 当前 v1.0 已完成 P0（2 个）+ P1（3 个）+ P2（3 个）共 8 个进阶功能。

---

## 一、离线学习包与资源管理

### 功能概述
考生在通勤、候考等无网络场景下需要离线学习资源，同时需要管理日益增长的本地存储。

### 功能清单

| 子功能 | 说明 | 复杂度 |
|--------|------|--------|
| 离线学习包 | 按科目/专项/真题年份打包下载，含题目、解析、知识点 | 中 |
| 增量更新 | 基于版本号或时间戳差量下载，节省流量 | 高 |
| 资源空间管理 | 各类资源占用展示 + 选择性清理 + 自动清理策略（N 天未用） | 低 |
| 统一收藏夹 | 题目/解析/时政素材/面试答案统一收藏，自定义文件夹/标签 | 中 |

### 技术依赖
- `path_provider` — 获取平台存储路径（已安装）
- `archive` 或 ZIP 压缩库 — 资源打包/解包
- SQLite 批量导入 — 离线包数据导入
- 增量同步协议 — 需要设计版本号管理方案

### 前置条件
- 需要资源分发服务器（CDN 或对象存储）
- 资源版本管理系统

### 建议实施方案
1. **Phase 1**（纯本地）：资源空间管理 + 统一收藏夹，不依赖后端
2. **Phase 2**（需服务端）：离线学习包下载 + 增量更新

## 二、多端同步与数据安全

### 功能概述
考生在手机和电脑之间切换学习，需要答题记录、错题本、收藏、学习计划等数据多端同步。同时增强隐私安全保护。

### 功能清单

| 子功能 | 说明 | 复杂度 |
|--------|------|--------|
| 多端数据同步 | 手机 ↔ 电脑实时同步，last-write-wins 冲突解决 | 高 |
| 本地备份/恢复 | 一键导出加密文件 + 从备份恢复 | 中 |
| WebDAV 云端备份 | 上传至用户自有网盘（坚果云、NextCloud 等） | 中 |
| 应用锁 | 指纹/密码/PIN 保护 | 低 |
| 数据加密 | 敏感数据（个人画像、API Key）本地加密增强 | 中 |
| 数据清除 | 一键清除所有个人数据（注销场景） | 低 |

### 同步范围
- `user_answers` — 答题记录
- `favorites` — 收藏
- `user_profile` — 用户画像
- `study_plans` + `daily_tasks` — 学习计划
- `mastery_scores` — 知识点掌握度
- `interview_sessions` + `interview_scores` — 面试记录
- `essay_submissions` — 申论作文
- `user_registrations` — 报名信息

### 技术依赖
- `webdav_client` — WebDAV 协议同步
- `encrypt` — AES 加密
- `local_auth` — 生物识别（指纹/面容）
- 同步协议：CRDTs 或 last-write-wins（推荐先用 last-write-wins，简单可靠）

### 建议实施方案
1. **Phase 1**（纯本地）：本地备份/恢复 + 应用锁 + 数据清除
2. **Phase 2**（WebDAV）：WebDAV 云端备份（用户自有网盘，无需自建服务器）
3. **Phase 3**（实时同步）：自建后端实时同步（需要用户系统 + 同步服务）

## 三、学习社区与组队备考

### 功能概述
提供同伴激励和经验交流平台，是投入最大、依赖最重的功能模块。

### 功能清单

| 子功能 | 说明 | 复杂度 |
|--------|------|--------|
| 备考小组 | 按考试/省份创建或加入小组，每日打卡排行榜 | 高 |
| 经验分享 | 上岸经验贴、题目讨论区、学习笔记分享 | 高 |
| AI 学习伙伴 | 虚拟备考同学角色，定期鼓励、模拟对答练习 | 中 |

### 技术依赖
- 云端后端（Supabase / 自建）：用户系统 + 内容存储 + 实时通信
- 内容审核：文本审核 API（敏感信息过滤）
- 实时通信：WebSocket 或 Supabase Realtime
- 文件存储：用户上传的笔记/图片

### 前置条件
- 用户注册/登录系统
- 后端 API 服务
- 内容审核机制
- 数据合规（用户生成内容存储、隐私协议）

### 建议实施方案
1. **Phase 1**（纯本地，可先做）：AI 学习伙伴 — 利用现有 LlmManager 实现虚拟备考同学角色，不依赖后端
2. **Phase 2**（需后端）：备考小组 + 打卡排行榜
3. **Phase 3**（需后端 + 审核）：经验分享 + 题目讨论区

## 后端选型建议

P3 功能中多端同步和学习社区需要后端支持，建议评估以下方案：

| 方案 | 优势 | 劣势 | 适合场景 |
|------|------|------|---------|
| **Supabase** | 开箱即用（Auth + DB + Realtime + Storage），免费额度 | 国内访问延迟，数据在海外 | 快速验证、小规模用户 |
| **自建后端（Dart Shelf/Go）** | 完全可控，国内部署 | 开发运维成本高 | 正式运营、大规模用户 |
| **WebDAV（仅同步）** | 零服务端成本，用户自有网盘 | 仅支持备份同步，无实时/社区 | 过渡方案 |

**推荐路径**：先做 WebDAV 备份（零成本验证需求）→ 用 Supabase 搭建社区 MVP → 用户量增长后迁移自建后端。

## 实施优先级排序

| 顺序 | 功能 | 理由 | 后端依赖 |
|------|------|------|---------|
| 1 | 本地备份/恢复 + 应用锁 + 数据清除 | 纯本地，用户刚需，零成本 | 无 |
| 2 | AI 学习伙伴 | 复用现有 LLM 能力，差异化体验 | 无 |
| 3 | 资源空间管理 + 统一收藏夹 | 纯本地，提升体验 | 无 |
| 4 | WebDAV 云端备份 | 低成本多端同步方案 | 无（用户自有网盘） |
| 5 | 离线学习包下载 | 需资源服务器 | CDN/对象存储 |
| 6 | 备考小组 + 打卡 | 需用户系统 | 后端 |
| 7 | 经验分享 + 讨论 | 需内容审核 | 后端 + 审核 |
| 8 | 实时多端同步 | 需自建同步服务 | 后端 |

> 其中 1-4 可在 v1.x 增量发布中实现（无后端依赖），5-8 需要在 v2.0 中伴随后端建设推进。
