# 考公考编智能助手 — 进阶功能设计文档

> 基于现有核心功能（题库刷题、模拟考试、人才引进匹配、学习路线、多模型 AI）之上的进阶扩展规划。

---

## 一、真题库收集与整理系统

### 1.1 需求背景
各地公务员、事业编考试每年出题风格和侧重点不同，考生需要针对目标地区的历年真题进行专项训练。目前题库缺乏按地区、年份维度的真题分类体系。

### 1.2 功能设计

#### 1.2.1 真题数据采集
- **数据来源**：
  - 官方公开真题（各省人事考试网历年公开试题）
  - 用户贡献（拍照/文字录入真题，AI 辅助结构化）
  - AI 回忆整理（结合网络信息 + LLM 能力还原真题）
- **覆盖范围**：
  - 国考（近 3 年）
  - 各省省考（近 3 年，优先覆盖热门省份：山东、江苏、浙江、广东等）
  - 事业编联考（近 3 年）
  - 选调生考试（近 3 年）

#### 1.2.2 真题结构化存储
- 新增字段维度：`region`（地区）、`year`（年份）、`exam_type`（国考/省考/事业编/选调）、`exam_session`（上半年/下半年）
- 每道真题关联：标准答案、官方解析、AI 深度解析、考点标签、难度系数
- 支持整套试卷还原（保留原始题序和分值分布）

#### 1.2.3 真题练习模式
- **按地区刷题**：选择目标省份 → 显示该省近 3 年所有真题
- **按年份刷题**：选择年份 → 横向对比各省同年考题
- **真题模考还原**：完全还原某年某省真题试卷，含原始时间限制和分值
- **跨省对比分析**：同一知识点在不同省份的出题频率和难度对比

#### 1.2.4 用户贡献真题
- 拍照上传 → OCR 识别 → AI 结构化（题干、选项、答案、解析）
- 文字粘贴 → AI 解析为标准题目格式
- 贡献审核机制：AI 初审（格式规范、答案合理性）+ 社区复审
- 贡献积分：用户上传真题获得积分，可兑换高级功能

### 1.3 技术要点
- `questions` 表扩展 `region`、`year`、`exam_type`、`exam_session`、`is_real_exam`（是否真题）字段
- OCR 集成：`google_mlkit_text_recognition`（本地）或云端 OCR API
- 真题去重：基于题干文本相似度（编辑距离 + LLM 语义判断）

## 二、AI 面试辅导系统

### 2.1 需求背景
公考面试（结构化面试/无领导小组讨论）是决定最终录取的关键环节，但市面辅导班价格高昂（数千到数万元）。利用 AI 可以提供低成本、高频次的面试模拟训练。

### 2.2 功能设计

#### 2.2.1 面试题库
- **结构化面试题库**：
  - 按题型分类：综合分析、计划组织、人际关系、应急应变、自我认知
  - 按地区分类：各省面试真题 + 高频模拟题
  - 按岗位分类：税务、海关、铁路公安等专业岗面试题
- **无领导小组讨论题库**：
  - 开放式讨论题、两难选择题、资源分配题、排序题
  - 附带讨论要点和参考框架

#### 2.2.2 AI 模拟面试
- **语音模拟模式**（进阶）：
  - 语音识别（STT）→ 用户口述回答 → 文字转写
  - AI 考官追问（根据回答内容动态生成追问）
  - 语音合成（TTS）→ AI 朗读题目和点评（可选）
- **文字模拟模式**（基础）：
  - AI 出题 → 用户文字作答 → AI 即时点评
  - 支持计时（每题思考 1 分钟 + 作答 3 分钟）
  - 模拟真实面试流程（3-4 题一组）
- **AI 考官角色扮演**：
  - 模拟不同风格考官（温和型/压力型/追问型）
  - 支持多轮追问，锻炼临场应变

#### 2.2.3 答题评分与反馈
- **多维度评分**：
  - 内容维度：观点深度、论据充分性、逻辑结构
  - 表达维度：条理清晰度、用语规范、口头禅检测（语音模式）
  - 时间维度：答题时长是否合理、各部分时间分配
- **详细批改报告**：
  - 逐段点评：指出优点和改进空间
  - 参考答案：给出高分答题框架和示范回答
  - 对比分析：用户答案 vs 参考答案的差距分析
- **历史面试记录**：
  - 所有模拟面试存档，支持回顾和重新练习
  - 面试能力雷达图（综合分析、组织协调、沟通表达、应变能力、自我认知）

#### 2.2.4 面试技巧知识库
- 结构化面试万能框架（审题→表态→分析→对策→总结）
- 各题型答题模板和常见踩坑点
- 面试礼仪指南（着装、坐姿、眼神、手势）
- 面试高频时政热点素材库

### 2.3 技术要点
- 语音识别：`speech_to_text` 插件（本地）或 Whisper API（云端）
- 语音合成：`flutter_tts`（本地 TTS 引擎）
- 面试评分 prompt 工程：设计多维度评分 rubric，确保评分一致性
- 新增数据表：`interview_questions`（面试题库）、`interview_sessions`（模拟面试记录）、`interview_scores`（评分详情）

## 三、AI 自适应智能出题

### 3.1 需求背景
传统题库按固定顺序或随机出题，无法根据考生水平动态调整。引入自适应算法，让每道题都精准匹配考生当前水平，最大化学习效率。

### 3.2 功能设计

#### 3.2.1 知识点掌握度建模
- 为每个知识点维护掌握度评分（0-100）
- 基于历史答题数据计算：正确率 × 时间衰减 × 题目难度权重
- 知识点关联图谱：前置知识点掌握不足时，自动回溯基础知识点

#### 3.2.2 智能出题策略
- **薄弱优先**：优先推送掌握度低于 60% 的知识点题目
- **遗忘曲线**：根据艾宾浩斯遗忘曲线，定期推送已掌握但可能遗忘的题目
- **难度递进**：同一知识点从简到难梯度推送
- **AI 动态生成**：当某知识点题库不足时，LLM 基于知识点和难度要求动态生成新题

#### 3.2.3 学习效率分析
- 每日学习效率曲线（正确率 × 答题速度的综合指标）
- 最佳学习时段分析（统计用户在不同时间段的表现）
- 预测达标日期：基于当前进度预测各科目达到目标分数的时间

### 3.3 技术要点
- 新增 `knowledge_points` 表（知识点 ID、名称、所属科目、前置知识点）
- 新增 `mastery_scores` 表（用户 × 知识点 × 掌握度 × 最后更新时间）
- 题目难度系数：初始人工标注，后续根据全局正确率自动校准
- AI 出题 prompt：指定知识点 + 难度 + 题型 → LLM 生成题目 → 质量校验

## 四、时政热点与申论素材库

### 4.1 需求背景
行测常识和申论写作高度依赖时政热点。考生需要持续跟进最新政策和社会热点，但信息分散、筛选困难。

### 4.2 功能设计

#### 4.2.1 时政热点自动汇编
- **数据来源**：
  - 人民日报、新华社等官媒 RSS/API
  - 政府工作报告、重要会议精神
  - 各省重要政策文件
- **AI 加工**：
  - 原文摘要（200 字以内）
  - 考试关联度评分（与行测常识/申论热点的关联性）
  - 考点提炼：可能出现的考题方向和考法
  - 申论可用角度：正反论述角度、常用论据

#### 4.2.2 申论素材库
- **按主题分类**：经济发展、社会治理、生态环保、文化教育、科技创新、乡村振兴等
- **素材类型**：
  - 名言金句（领导人讲话、经典引用）
  - 典型案例（正面/反面事例）
  - 政策表述（规范用语、官方提法）
  - 数据支撑（关键统计数据）
- **智能推荐**：根据用户练习的申论题目主题，自动推荐相关素材

#### 4.2.3 申论写作训练
- **分段训练**：开头、分论点、结尾分别训练，AI 逐段批改
- **全文写作**：限时完成完整申论，AI 按评分标准打分
- **范文对比**：用户作文 vs AI 生成范文，逐段对比差距
- **高频主题专练**：近 3 年申论高频主题集中训练

### 4.3 技术要点
- 新增 `hot_topics` 表（标题、摘要、来源、发布日期、关联度、考点）
- 新增 `essay_materials` 表（类型、主题、内容、来源）
- 新增 `essay_submissions` 表（用户申论作文 + AI 批改结果）
- RSS 解析：`webfeed` 包 + Dio 定时拉取
- 素材检索：SQLite FTS（全文搜索）或关键词索引

## 五、考试日历与报名管家

### 5.1 需求背景
公考考生通常同时关注多个考试（国考、省考、事业编、选调），报名时间、缴费截止、准考证打印、笔试面试时间散落各处，容易遗漏。

### 5.2 功能设计

#### 5.2.1 考试日历
- 全国/分省考试日历视图（月视图 + 列表视图）
- 数据来源：AI 抓取各省人事考试网公告 + 人工维护
- 支持订阅：用户选择关注的考试类型和省份，日历只显示相关考试
- 关键时间节点：公告发布 → 报名开始 → 报名截止 → 缴费截止 → 准考证打印 → 笔试 → 成绩公布 → 面试

#### 5.2.2 报名提醒
- 多级提醒：报名倒计时 7/3/1 天、缴费截止前 3/1 天
- 与学习计划联动：自动在考试日历基础上生成备考倒计时
- 报名资格预检：根据用户画像自动判断是否满足报名条件

#### 5.2.3 报名信息管理
- 存储用户已报名考试的信息（准考证号、考场地址、考试时间）
- 考试当天提醒清单（证件、文具、考场路线）
- 考后成绩查询入口（自动跳转对应查询页面）

### 5.3 技术要点
- 新增 `exam_calendar` 表（考试名称、类型、省份、各时间节点、状态）
- 新增 `user_registrations` 表（用户报名记录）
- 本地通知：`flutter_local_notifications`
- 日历组件：`table_calendar` 或自定义日历 Widget

## 六、错题深度分析与知识图谱

### 6.1 需求背景
当前错题本只是简单收录错题，缺乏对错误模式的深度分析。考生需要知道"为什么总错"以及"还有哪些关联知识点也薄弱"。

### 6.2 功能设计

#### 6.2.1 错题模式分析
- **错误类型标注**：AI 分析每道错题的错因类别
  - 知识盲区（完全不会）
  - 概念混淆（相似知识点混淆）
  - 粗心大意（会做但做错）
  - 时间不足（没来得及做）
  - 陷阱题（被干扰项误导）
- **错因分布统计**：各类错因的占比图表
- **高频错误知识点 TOP 10**

#### 6.2.2 知识图谱可视化
- 以科目为根节点，知识点为子节点，构建树状/网状图谱
- 节点颜色表示掌握度（绿色已掌握 → 红色薄弱）
- 点击节点可直接进入该知识点的专项练习
- 知识点间的关联关系可视化（前置依赖、相似易混）

#### 6.2.3 AI 错题诊断报告
- 每周自动生成错题诊断报告
- 内容：本周薄弱知识点、进步知识点、建议下周重点
- 智能归因：区分"知识问题"和"应试技巧问题"，给出针对性建议

### 6.3 技术要点
- 扩展 `user_answers` 表增加 `error_type` 字段
- 知识图谱可视化：`graphview` 或自定义 Canvas 绘制
- AI 错因分析 prompt：输入题目+用户答案+正确答案 → 输出错因类别+分析

## 七、学习社区与组队备考

### 7.1 需求背景
备考是一个漫长且孤独的过程，考生需要同伴激励和经验交流。同时，优质学习笔记和经验分享可以帮助更多考生。

### 7.2 功能设计

#### 7.2.1 备考小组
- 按目标考试/省份创建或加入备考小组
- 小组内打卡：每日学习时长和刷题量排行榜
- 小组目标：共同设定目标分数，互相监督

#### 7.2.2 经验分享
- 上岸经验贴（已考上的用户分享备考经验）
- 题目讨论区（对有争议的题目进行讨论）
- 学习笔记分享（用户整理的知识点笔记）

#### 7.2.3 AI 学习伙伴
- 虚拟学习伙伴角色（AI 模拟一个同样在备考的"同学"）
- 定期鼓励和提醒
- 模拟对答练习（AI 扮演考友互相出题）

### 7.3 技术要点
- 此功能需要云端后端支持（用户系统、内容存储、实时通信）
- 建议作为 v2.0 版本的核心功能
- 初期可先实现本地的"AI 学习伙伴"功能，不依赖后端

## 八、个性化数据看板

### 8.1 需求背景
考生需要一个全局视角了解自己的备考状态，而不是分散在各个功能模块中查看零散数据。

### 8.2 功能设计

#### 8.2.1 综合仪表板
- **今日概览**：今日已学时长、刷题量、正确率、距下次考试天数
- **能力雷达图**：行测五大模块 + 申论 + 公基的综合能力值
- **学习热力图**：类 GitHub 贡献热力图，展示每日学习强度（连续打卡天数）
- **备考进度条**：总体备考完成度（基于学习计划）

#### 8.2.2 对比分析
- **自我对比**：本周 vs 上周各维度数据对比
- **目标差距**：当前能力 vs 目标岗位要求的差距可视化
- **预测分析**：基于当前趋势预测考试日的预估分数

#### 8.2.3 学习报告
- **周报**：每周日自动生成，总结本周学习数据和下周建议
- **月报**：每月进步趋势、薄弱项变化、学习效率分析
- **支持导出**：PDF/图片格式导出，便于分享或打印

### 8.3 技术要点
- 雷达图/热力图：`fl_chart` 或 `syncfusion_flutter_charts`
- 学习时长统计：记录每次刷题/看题的时间区间，汇总计算
- PDF 导出：`pdf` + `printing` 包

## 九、离线学习包与资源管理

### 9.1 需求背景
考生在通勤、候考等场景下可能没有网络，需要提前下载学习资源。同时，大量题库和素材会占用存储空间，需要管理。

### 9.2 功能设计

#### 9.2.1 离线学习包
- 按科目/专项/真题年份打包下载
- 包含题目、解析、知识点说明
- 增量更新：只下载变更部分，节省流量

#### 9.2.2 资源空间管理
- 显示各类资源占用的存储空间
- 支持选择性清理（已完成的旧试卷、过期的时政素材）
- 自动清理策略：超过 N 天未使用的缓存资源自动清理

#### 9.2.3 学习资料收藏夹
- 收藏题目、解析、时政素材、面试答案到统一收藏夹
- 分类管理：自定义文件夹/标签
- 收藏内容支持离线访问

### 9.3 技术要点
- 资源打包：ZIP 压缩 + SQLite 批量导入
- 存储管理：`path_provider` 获取路径 + 文件系统 API 统计大小
- 增量同步：基于版本号或时间戳的差量下载

## 十、多端同步与数据安全

### 10.1 需求背景
考生可能在手机和电脑之间切换学习，需要学习进度、错题本、收藏等数据在多设备间同步。

### 10.2 功能设计

#### 10.2.1 多端数据同步
- 手机 ↔ 电脑的学习进度实时同步
- 冲突解决策略：以最后修改时间为准，冲突时保留两份
- 同步范围：答题记录、错题本、收藏、学习计划进度、用户画像

#### 10.2.2 数据备份与恢复
- 本地备份：一键导出所有数据为加密文件
- 云端备份：可选上传至用户自己的网盘（WebDAV 协议）
- 恢复功能：从备份文件恢复全部数据

#### 10.2.3 隐私安全增强
- 应用锁：指纹/密码保护
- 数据加密：敏感数据（个人画像）本地加密存储
- 数据清除：一键清除所有个人数据（注销场景）

### 10.3 技术要点
- 同步协议：基于 CRDTs 或简单的 last-write-wins 策略
- WebDAV 集成：`webdav_client` 包
- 加密：`encrypt` 包进行 AES 加密
- 应用锁：`local_auth`（生物识别）

## 实施优先级建议

| 优先级 | 功能模块 | 理由 | 依赖 |
|--------|---------|------|------|
| P0 | 真题库收集与整理 | 真题是备考最核心的资源，直接提升产品价值 | 题库系统已就绪 |
| P0 | AI 面试辅导（文字模式） | 填补面试备考空白，差异化竞争力强 | LLM 抽象层已就绪 |
| P1 | 时政热点与申论素材库 | 高频使用场景，提升日活 | RSS 解析 + LLM |
